# 仓库克隆说明

## 克隆位置

代码仓库克隆到 `yudao-deploy` 的上一级目录的 `github` 文件夹中，即相对于 `yudao-deploy` 目录的 `../../github/`。

```
yudao-deploy/          # 当前部署配置项目
../github/             # 代码仓库（相对于 yudao-deploy 的 ../github/）
```

## 需要克隆的仓库

### 1. 后端 - ruoyi-vue-pro

```bash
# 在 github 目录下克隆
cd ../../github
git clone -b master-jdk17 https://github.com/YunaiV/ruoyi-vue-pro.git
```

### 2. 前端 - yudao-ui-admin-vue3

```bash
# 在 github 目录下克隆
cd ../../github
git clone -b master https://github.com/yudaocode/yudao-ui-admin-vue3.git
```

## 克隆后的目录结构

```
├── github/                            # 代码仓库（yudao-deploy 的兄弟目录）
│   ├── ruoyi-vue-pro/                # 后端代码
│   │   └── yudao-server/
│   │       └── target/
│   │           └── yudao-server.jar
│   └── yudao-ui-admin-vue3/          # 前端代码
│       └── Dockerfile
│
└── yudao-deploy/                      # 部署配置
    ├── docker-compose.server.yml
    └── frontend/
        └── nginx.conf
```

## 配置说明

`docker-compose.server.yml` 中的构建路径已使用相对路径：

```yaml
server:
  build:
    context: ../../github              # 指向 ../github/
    dockerfile: ruoyi-vue-pro/backend/Dockerfile

frontend:
  build:
    context: ../../github              # 指向 ../github/
    dockerfile: yudao-ui-admin-vue3/Dockerfile
```

这样无论在 Windows 还是 Mac 上，只要目录结构一致（yudao-deploy 与 github 是兄弟目录），配置都能正常工作。
